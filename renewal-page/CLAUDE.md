# renewal-page 프로젝트 지침

이미지 기반 HTML 프로토타입 자동 생성 프로젝트

---

## 1. 프로젝트 개요

### 목표
- 주어진 스크린샷을 분석하여 **동일한 레이아웃, 색상, 폰트의 HTML 페이지** 생성
- 반복 수정을 통해 인터랙티브한 웹 경험 제공
- 빠른 프로토타입 개발

### 기술 스택
- **Frontend**: Vanilla HTML + CSS + JavaScript (라이브러리 최소화)
- **No Build Tool**: 직접 브라우저에서 열어 확인 가능

---

## 2. 프로젝트 구조

각 변환 요청마다 **독립된 폴더**를 생성합니다:

```
renewal-page/
├── CLAUDE.md (이 파일)
├── projects/
│   ├── [프로젝트명1]/
│   │   ├── input/
│   │   │   └── screenshot.png (원본 이미지)
│   │   ├── output/
│   │   │   ├── index.html
│   │   │   ├── style.css (필요시)
│   │   │   └── script.js (필요시)
│   │   └── spec.md (변환 사양 및 요구사항)
│   └── [프로젝트명2]/
│       ├── input/
│       ├── output/
│       └── spec.md
└── docs/
    └── (참고 문서)
```

### 폴더명 규칙
- 형식: `[목적]-[구분명]` 또는 `[페이지명]`
- 예시: `login-page`, `dashboard-v2`, `card-component`

---

## 3. 요구사항 제공 형식

사용자가 변환을 요청할 때 **다음 정보를 함께 제공**해야 합니다:

### 필수 정보
```
📁 폴더명: [프로젝트 이름]
📸 이미지: [스크린샷 파일]
📝 spec.md에 포함될 내용:
  - 페이지 목적/설명
  - 주요 색상 코드 (HEX/RGB)
  - 사용 폰트 정보 (또는 추측)
  - 인터랙션 요구사항
  - 특이사항 (애니메이션, 반응형 등)
```

### spec.md 예시
```markdown
# [프로젝트명] 변환 사양

## 요구사항
- 목적: 로그인 페이지 프로토타입
- 주요 색상:
  - 메인: #2563EB (파란색)
  - 배경: #F3F4F6 (연한 회색)
  - 텍스트: #1F2937 (진한 회색)
- 폰트: 시스템 기본 (또는 Google Fonts)

## 기능
- 이메일, 비밀번호 입력 필드
- "로그인" 버튼 (클릭 시 경고창)
- "회원가입" 링크

## 수정사항 (추가로 요청할 때)
- [ ] 변경사항 1
- [ ] 변경사항 2
```

## 4. HTML 생성 프로세스

### 단계 1: 분석
- 이미지에서 레이아웃 구조 파악 (헤더, 본문, 푸터)
- 색상, 폰트, 간격 측정/추정
- 인터랙티브 요소 식별

### 단계 2: HTML 작성
```html
<!-- 기본 구조 -->
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>페이지 제목</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- 내용 -->
  <script src="script.js"></script>
</body>
</html>
```

- **의미있는 HTML**: `<div>` 남용 금지, `<header>`, `<main>`, `<button>` 등 시맨틱 태그 사용
- **클래스명**: BEM 또는 직관적 이름 (예: `btn-primary`, `card-container`)
- **접근성**: alt 텍스트, ARIA 레이블 고려

### 단계 3: CSS 작성
- Flexbox/Grid 활용하여 레이아웃 구성
- 원본 이미지와 색상, 간격, 텍스트 크기 최대한 일치
- 반응형 디자인 (모바일, 태블릿, 데스크톱)

### 단계 4: JavaScript (필요시)
- 이벤트 핸들러 (버튼 클릭, 입력 필드 등)
- 간단한 유효성 검사
- DOM 조작 (선택적)

## 5. 품질 기준

### 레이아웃 정확성
- ✅ 원본 이미지와 **구조적으로 동일**
- ✅ 여백, 정렬, 순서 일치
- ❌ 픽셀 완벽 일치는 불필요 (개념적 일치)

### 색상 및 스타일
- ✅ 주요 색상 정확한 매칭
- ✅ 폰트 크기, 굵기 비슷한 수준
- ✅ 보더, 그림자, 배경색 구현

### 인터랙션
- ✅ 호버, 클릭 효과 구현
- ✅ 입력 필드 동작 확인
- ✅ 필요한 유효성 검사 포함

---

## 6. 수정 및 반복

### 요구사항 변경 시
1. spec.md에 변경사항 기록
2. 해당 파일(HTML/CSS/JS) 수정
3. 수정 후 결과를 사용자에게 확인 요청

### 버전 관리
- 주요 변경 시 폴더명에 버전 추가 (예: `login-page-v2`)
- 또는 output/ 폴더에 날짜 포함된 파일명 사용 (예: `index-20260327.html`)

---

## 7. 체크리스트

HTML 생성 후 반드시 확인:

- [ ] 브라우저에서 열어 화면 확인
- [ ] 원본 이미지와 구조/색상 비교
- [ ] 모든 텍스트가 정확하게 표시됨
- [ ] 클릭 가능한 요소에 마우스 오버 효과
- [ ] 반응형 확인 (드래그로 창 크기 조정)
- [ ] 콘솔 에러 없음 (F12 개발자 도구)

---

## 8. 특수 상황

### 폰트 파악 어려울 때
- 시스템 기본 폰트 사용: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- 필요시 Google Fonts 추가 (예: `Poppins`, `Noto Sans KR`)

### 색상 정확도 필요시
- 온라인 컬러 픽커 도구로 이미지 분석
- HEX 코드 명시

### 이미지/아이콘 필요시
- SVG 또는 간단한 CSS 도형 우선
- 외부 라이브러리 최소화
